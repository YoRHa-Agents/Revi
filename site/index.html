Revi is a review tool where human feedback meets AI action. Anchored comments on plans, designs, and prototypes — structured JSON exports that agents consume, act on, and resolve.

Features

Everything for the review loop

Three content types, six anchor types, one clean API. Revi keeps humans and agents in sync.

📄

Plan Review

Markdown documents with heading anchors, text selection comments, inline search, and a collapsible document index.

🎨

Design Review

Images on a dark canvas with numbered annotation pins. Click a pin to comment; hover a card to highlight it.

🖥️

Prototype Review

Step-through interactive HTML prototypes with per-step commenting, zoom, and pan support.

🤖

Agent Export API

A single endpoint returns only open comments as structured JSON — purpose-built for LLM agent consumption.

💬

Anchored Comments

Six reference types — section, quote, line, annotation, step, general — so every comment knows where it points.

⚡

Zero-Dependency Binary

Single static Rust binary. No runtime, no containers, no config. Download and run.

Workflow

The review-revise loop

A continuous cycle: humans flag issues, agents fix them, humans verify.

1

Drop files

Place markdown plans, design images, or HTML prototypes into the workspace folder.

2

Review & comment

Human reviewers browse in the split-pane UI and leave anchored comments on specific sections.

3

Agent reads export

AI agents poll /api/export/{id} to get only open comments with anchor references.

4

Act & resolve

Agents process feedback, make changes, and mark comments resolved via the API.

For Agents

One endpoint, structured feedback

The export API returns only what agents need — open comments with anchors and metadata.

agent-export.json
{
  "schemaVersion": "1.0",
  "item": {
    "id": "plans/sprint-1",
    "type": "plan",
    "title": "Sprint 1 Design"
  },
  "openComments": [{
    "author": "Alice",
    "content": "Add error handling",
    "reference": {
      "type": "section",
      "value": "## API Layer"
    }
  }]
}

Built for automation

Every comment carries a typed reference so agents know exactly where to look and what to fix.

  • section — markdown heading anchor
  • quote — inline text selection
  • line — line number
  • annotation — image pin number
  • step — prototype step index
  • general — whole-item comment

Quick Start

Running in 30 seconds

terminal
# Download and run — zero dependencies
./dist/revi-linux-x86_64

# Start the frontend
cd frontend && npm install && npm run dev

# Open http://localhost:5173

# Or with custom paths:
./revi --workspace /my/docs --data /my/data --port 9000

Two backends, your choice

A production-ready Rust binary (zero-dependency, statically linked) or a Python FastAPI server for development.

  • Rust binary auto-creates workspace and data dirs
  • Config via CLI flags, TOML file, or defaults
  • Python backend for quick prototyping
  • Vue 3 SPA with Vite hot-reload
